네이티브 SVG 애니메이션

SVG 파일 안에 직접 들어가는 <animate> 계열 태그. 외부 CSS·JS 없이 그 자체로 재생됩니다. 자기완결형 아이콘·인포그래픽에 강하지만, 복잡한 시퀀스나 정밀 제어에는 약합니다.

속성 변화

<animate>

코드 보기
<circle r="14">
  <animate attributeName="r"
    values="10;26;10"
    dur="1.6s"
    repeatCount="indefinite"/>
</circle>

회전 · 변형

<animateTransform>

코드 보기
<animateTransform
  attributeName="transform"
  type="rotate"
  from="0 40 40"
  to="360 40 40"
  dur="1s" repeatCount="indefinite"/>

경로 따라 이동

<animateMotion>

코드 보기
<circle r="6">
  <animateMotion dur="2s"
    repeatCount="indefinite"
    path="M12 62 Q40 6 68 62"/>
</circle>

형태 모핑

<animate d>

코드 보기
// 점 개수가 같은 두 path를 오감
<animate attributeName="d"
  dur="2.6s" repeatCount="indefinite"
  values="M40 16 …Z;
          M40 12 …Z;
          M40 16 …Z"/>
엔진 · CSS

CSS로 제어하기

가장 널리 쓰이는 방식. @keyframes와 transition으로 SVG 요소를 제어합니다. transform·opacity는 GPU 가속이 되어 성능이 가장 좋습니다.

키프레임 루프

@keyframes

코드 보기
.dot{animation:wave 1.1s infinite}
.dot:nth-child(2){animation-delay:.15s}
.dot:nth-child(3){animation-delay:.3s}
@keyframes wave{
  30%{transform:translateY(-11px)}
}

선 그리기

stroke-dashoffset

코드 보기
path{
  stroke-dasharray: 320;   /* 선 길이 */
  stroke-dashoffset: 320;  /* 숨김 */
  animation: draw 1.5s ease forwards;
}
@keyframes draw{ to{stroke-dashoffset:0} }

전환 · 호버

transition

↑ 카드 위에 마우스 올리기
코드 보기
.icon{transition:transform .3s ease}
.card:hover .icon{
  transform:translateX(8px)
}
엔진 · JavaScript

라이브러리로 정밀 제어

타임라인, 스크롤 연동, 스프링/물리 효과처럼 복잡한 시퀀스가 필요할 때. JS는 제어력이 가장 높습니다. 아래는 대표 도구이고, 다음 섹션의 인터랙션들은 모두 순수 JavaScript로 제어됩니다.

GSAP JS

업계 표준 타임라인 엔진. 시퀀스·ScrollTrigger·MorphSVG 플러그인까지, 가장 강력하고 안정적.

anime.js JS

가볍고 직관적. SVG path 모핑·라인 드로잉·스태거(staggering)에 특히 간편.

Vivus JS

SVG 선 그리기 전용 라이브러리. stroke 애니메이션을 한 줄로 처리.

Lottie JSON

After Effects 모션을 JSON으로 내보내 웹·앱에서 재생. 복잡한 일러스트 모션의 사실상 표준.

Web Animations API 네이티브

브라우저 내장 element.animate(). 라이브러리 없이 JS로 키프레임 제어.

Framer Motion React

React 컴포넌트용. 선언적 props로 스프링·제스처·레이아웃 애니메이션.

언제 무엇을 쓸까

같은 결과도 엔진에 따라 비용과 한계가 다릅니다. 단순할수록 SMIL·CSS, 복잡할수록 JS로.

엔진제어력성능복잡한 시퀀스대표 용도
SMIL보통좋음약함자기완결형 아이콘, 인포그래픽, 단순 로더
CSS중상매우 좋음보통호버·로더·선 그리기·마이크로 인터랙션
JavaScript매우 높음좋음(최적화 필요)강함스크롤 트리거, 타임라인, 물리/제스처, 복잡 모핑
접근성 한 줄. 끊임없이 반복되는 모션은 멀미·주의분산을 유발할 수 있습니다. @media (prefers-reduced-motion: reduce)를 존중해, 모션 감소 설정을 켠 사용자에게는 자동 재생을 멈추는 것이 좋습니다. (이 페이지도 해당 설정 시 일시정지로 시작합니다.)

핵심 좌표 — 변형은 transform/opacity 위주로(GPU 가속), 선 그리기는 stroke-dasharray+stroke-dashoffset, 형태 변형은 같은 점 개수의 path 사이 모핑.
