1. CSS 애니메이션의 종류

실무에서는 “무엇이 애니메이션을 시작시키는가”와 “시간을 누가 제어하는가”로 분류하면 설계가 쉬워집니다.

State change

Transition

hover, focus, checked, class 변경처럼 두 상태 사이를 자연스럽게 보간합니다. 버튼, 카드, 드롭다운, 토글에 가장 많이 씁니다.

Timeline

@keyframes Animation

0%→100% 단계가 있는 독립적인 시간표입니다. 로더, 스켈레톤, 알림 뱃지, 배경 움직임처럼 자동 실행/반복 UI에 적합합니다.

Input-linked

Scroll-driven

시간이 아니라 스크롤 위치가 진행률이 됩니다. 읽기 진행바, 패럴랙스, 섹션 등장, 제품 스토리텔링에 좋습니다.

Media

Image / Video UI

이미지 zoom, reveal, mask, gallery, 영상 오버레이, progress, subtitle, reel 전환처럼 콘텐츠 몰입도를 높입니다.

Navigation

View Transition

목록→상세, 라우트 이동, 페이지 전환에서 이전/다음 뷰를 스냅샷으로 연결해 화면이 끊기지 않게 합니다.

Drawing

SVG / Stroke

아이콘 체크, 로고 드로잉, 경로 애니메이션에 사용합니다. stroke-dasharray와 stroke-dashoffset 조합이 핵심입니다.

Transition vs Keyframes 체감

왼쪽은 hover로 상태가 변할 때만 움직이고, 오른쪽은 독립적으로 계속 재생됩니다.

Transition: 상품 카드 hover

실제 적용: 커머스 카드, 추천 상품, 가격/장바구니 CTA.

Air Desk Lamp

hover하면 lift, shadow, 내부 이미지 각도가 동시에 전환됩니다.

@keyframes: 로딩/상태 표시

실제 적용: 결제 처리, 파일 업로드, 데이터 fetching.

CSS

Transition 기본형

두 상태를 정의하고, 원래 상태에 transition을 둡니다.

.card {
  transform: translateY(0) scale(1);
  transition:
    transform 320ms cubic-bezier(.2, 1.4, .38, 1),
    box-shadow 320ms ease;
}
.card:hover {
  transform: translateY(-10px) scale(1.025);
  box-shadow: 0 28px 60px rgba(0,0,0,.24);
}

@keyframes 기본형

반복, 방향, 지연, fill-mode까지 제어할 수 있습니다.

@keyframes spin {
  to { transform: rotate(360deg); }
}
.loader {
  animation: spin .95s linear infinite;
}

2. 이미지에 자주 쓰는 CSS 애니메이션

이미지는 layout을 흔들지 않도록 transform, opacity, clip-path, mask 계열로 다루는 것이 일반적입니다.

이미지 zoom + caption reveal

실제 적용: 여행/숙소 카드, 매거진 썸네일, 포트폴리오 프로젝트 카드.

Gallery thumbnail transition

실제 적용: 상세페이지 이미지 갤러리, 룩북, 부동산/호텔 사진 탐색.

clip-path mask reveal

실제 적용: before/after 비교, 기능 공개, 캠페인 랜딩 인터랙션.

Hover / Move cursor

마우스 위치를 CSS 변수로 받아 원형 마스크를 확장합니다.

이미지 애니메이션 설계 포인트

이미지를 직접 width/height로 키우기보다, wrapper는 고정하고 내부 이미지만 transform합니다.

.image-card { overflow: hidden; }
.image-card img {
  transform: scale(1);
  transition: transform 900ms cubic-bezier(.2,.8,.2,1);
}
.image-card:hover img { transform: scale(1.12); }
.caption {
  transform: translateY(18px);
  opacity: 0;
  transition: transform 350ms, opacity 350ms;
}
.image-card:hover .caption {
  transform: translateY(0);
  opacity: 1;
}

영상 overlay 패턴

재생 상태 class 하나로 play button, progress, subtitle을 동시에 제어합니다.

.video.is-playing .play-button {
  transform: scale(.72);
  opacity: 0;
}
.video.is-playing .progress span {
  animation: videoProgress 8s linear infinite;
}
@keyframes videoProgress {
  from { width: 0%; }
  to   { width: 100%; }
}

Preview hover 패턴

목록 카드에서는 hover/focus 때만 영상 미리보기 레이어를 켜는 식으로 비용을 줄입니다.

.video-card .preview {
  opacity: 0;
  transform: scale(1.04);
  transition: opacity 220ms, transform 600ms;
}
.video-card:hover .preview,
.video-card:focus-within .preview {
  opacity: 1;
  transform: scale(1);
}

View Transition 최소 예시

라우터나 상태 업데이트 함수를 callback으로 감싸는 구조입니다.

function updateView() {
  app.dataset.route =
    app.dataset.route === 'home' ? 'detail' : 'home';
}

if (!document.startViewTransition) {
  updateView();
} else {
  document.startViewTransition(updateView);
}

01. Problem

스크롤 기반 애니메이션은 긴 랜딩 페이지에서 사용자가 어디까지 읽었는지, 다음 내용이 무엇인지 알려주는 안내 장치로 쓰입니다.

02. Product

제품의 구조나 기능을 단계적으로 보여줄 때, 요소가 한 번에 튀어나오는 것보다 viewport 진입에 맞춰 등장하는 편이 자연스럽습니다.

03. Result

마지막 CTA 근처에서는 과한 모션보다 명확한 focus, contrast, button feedback이 더 중요합니다.

실무 체크리스트

애니메이션을 넣기 전, 아래 질문에 답하면 품질이 크게 올라갑니다.

1. 목적이 있는가?

상태 변화, 방향 안내, 계층 강조, 피드백 중 하나를 명확히 합니다. 목적 없는 배경 움직임은 가장 먼저 줄입니다.

2. transform/opacity로 가능한가?

위치 이동은 top/left보다 translate, 크기 변화는 width/height보다 scale을 먼저 검토합니다.

3. 시간은 짧은가?

마이크로 인터랙션은 대개 150~300ms, 모달/카드 전환은 240~450ms, 히어로/스크롤 연출은 600ms 이상도 가능하지만 반복을 줄입니다.

4. 접근성을 반영했는가?

prefers-reduced-motion, pause/stop, focus 상태, hover 없는 터치 환경을 고려합니다.

5. fallback이 있는가?

scroll-driven, view transition처럼 최신 기능은 @supports나 JS feature detection으로 일반 동작을 보장합니다.

6. 운영/성능 검증을 했는가?

DevTools Performance, CPU throttling, low-end mobile에서 jank가 없는지 확인합니다.
