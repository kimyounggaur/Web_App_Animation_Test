## SVG 애니메이션을 보는 두 개의 축

헷갈리지 않으려면 "무엇으로 구동하나(엔진)"와 "무엇을 움직이나(기법)"를 나눠서 보면 됩니다. 같은 결과도 엔진 조합이 다를 수 있어요.

**엔진(구동 방식) 3종.** 첫째 **SMIL**은 SVG 안에 직접 박히는 `<animate>`·`<animateTransform>`·`<animateMotion>` 태그로, 외부 CSS·JS 없이 파일 그 자체로 재생됩니다(파일 1번 섹션). 모든 최신 브라우저에서 동작하지만 — 한때 Chrome이 폐기를 예고했다가 철회한 이력이 있고 — 복잡한 제어에는 약합니다. 둘째 **CSS**는 `@keyframes`와 `transition`으로 SVG 요소를 움직이는 가장 보편적인 방식이고(2번 섹션), `transform`·`opacity` 기반이라 성능이 가장 좋습니다. 셋째 **JavaScript**는 GSAP·anime.js 같은 라이브러리나 내장 `element.animate()`로 타임라인·스크롤·물리까지 정밀 제어합니다.

**기법(무엇이 변하나) 5종.** 이게 흔히 말하는 "애니메이션의 종류"입니다.

- **속성 변화** — `r`, `cx`, `fill`, `opacity` 같은 값을 보간. → 맥동하는 점, 색 전환, 깜빡임
- **변형(transform)** — `translate`/`rotate`/`scale`. → 스피너, 슬라이드 인, 확대 팝
- **경로 이동** — `<animateMotion>` 또는 CSS `offset-path`. → 곡선을 따라 흐르는 점, 지도 위 이동 마커
- **선 그리기(stroke)** — `stroke-dasharray` + `stroke-dashoffset`. → 로고·서명·체크가 스스로 그려지는 효과
- **형태 모핑(morphing)** — `path`의 `d` 값을 보간. → 햄버거 ↔ X, 재생 ↔ 정지, 블롭 변형, 브랜드 전환

## 실제 적용사례 (기법 → 제품 패턴)

파일 4번 섹션의 인터랙션들이 바로 이 매핑을 라이브로 보여줍니다.

| 어디서 보나 | 핵심 기법 | 주 엔진 |
|---|---|---|
| 로딩 스피너 / 버퍼링 | transform 회전 | CSS · SMIL |
| 업로드·다운로드 진행률(원형/막대) | stroke-dashoffset | CSS + JS |
| 결제·제출 "성공" 체크 | stroke 그리기(원→체크 순차) | CSS · JS |
| 햄버거 ↔ 닫기, 재생 ↔ 정지 | transform 모핑·토글 | CSS |
| 좋아요·북마크 "팝" | scale + 색 transition | CSS |
| 로고·서명 인트로(그려지기) | stroke 그리기 | CSS · Vivus |
| 데이터 시각화(막대·선이 자라남) | 속성/transform 보간 | JS(D3·GSAP) |
| 랜딩 히어로·온보딩 일러스트 | 복합 모션 | Lottie · GSAP |
| 스크롤 연동 스토리텔링 | 스크롤 진행도→애니메이션 | JS(ScrollTrigger) |

특히 **로더·진행률·성공 체크·아이콘 토글**은 거의 모든 앱·웹에 들어가는 4대장이고, **로고 드로잉**과 **온보딩 일러스트(Lottie)**는 브랜드 경험에서 가장 눈에 띄는 활용처입니다.

## 가장 많이 쓰는 두 기법의 원리

**선 그리기** — `path`의 전체 길이 L을 구해 `stroke-dasharray: L; stroke-dashoffset: L`로 선 전체를 "빈 칸"처럼 숨긴 뒤, offset을 `0`으로 보간하면 선이 그어지는 것처럼 보입니다. 정확한 L은 JS의 `getTotalLength()`로 측정합니다(파일의 별·체크가 이 방식).

**형태 모핑** — 출발 `path`와 도착 `path`의 앵커 점 개수와 명령 구조가 같아야 부드럽게 보간됩니다. 다르면 끊기므로, 이럴 땐 GSAP MorphSVG나 flubber 같은 도구가 점 개수를 자동으로 맞춰줍니다.

## 엔진 선택 요약

자기완결형 아이콘·인포그래픽이면 SMIL이나 CSS, UI 마이크로 인터랙션 대부분은 CSS(성능·간결함), 타임라인·스크롤·물리·복잡 모핑은 JS, 디자이너가 After Effects로 만든 복잡한 모션 일러스트는 Lottie가 정답에 가깝습니다. 성능의 핵심은 한 줄로, `transform`/`opacity`만 애니메이션하면 GPU 합성으로 60fps가 쉽지만 `width`·`top` 같은 레이아웃 속성은 리플로우를 일으켜 끊깁니다.

마지막으로, 이런 패턴들이 실제 화면에서 어떤 느낌인지 참고 이미지도 몇 장 가져와 보겠습니다.위 이미지들이 보여주듯 실제 제품에서는 로더·버튼 상태·성공 피드백 같은 마이크로 인터랙션으로 가장 자주 등장하는데, 정지 이미지로는 움직임이 안 보이니 **실제 모션은 위 HTML 파일에서 직접 돌려보는 게** 가장 정확합니다. 특정 기법(예: 모핑이나 스크롤 트리거)을 GSAP·anime.js 코드 예제까지 더 깊게 파고들고 싶으면 그 부분만 따로 짚어드릴게요.
