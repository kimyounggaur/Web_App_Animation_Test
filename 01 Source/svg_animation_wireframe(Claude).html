SVG 애니메이션을
실제 UI로 이해하기

로고가 그려지고, 선이 경로를 따라 움직이고, 카드 결제 버튼이 로딩에서 성공/오류 상태로 바뀌는 흐름을 한 페이지에서 테스트합니다.

SGV라고 쓰신 부분은 SVG로 해석했습니다. SVG는 DOM 요소라서 CSS, SMIL, JavaScript, 디자인 툴 내보내기 방식으로 모두 움직일 수 있습니다.

한눈에 보는 선택 기준

SVG 애니메이션은 “어디를 움직일 것인가”보다 “누가 제어할 것인가”로 분류하면 실무 판단이 빨라집니다.

방식가장 잘 맞는 상황주의점
CSS아이콘, 버튼, 로더, hover/focus, 선 그리기복잡한 타임라인·상태 분기는 어렵습니다.
SMILHTML 없이 SVG 파일만 열어도 재생되는 단순 반복팀 표준이 React/JS라면 관리 방식이 분리될 수 있습니다.
JavaScript클릭, 스크롤, 데이터 변화, 조건부 상태 전환DOM 조작이 많으면 성능 설계가 필요합니다.
LottieAfter Effects 기반 복잡한 일러스트/온보딩플레이어, 접근성 텍스트, 파일 용량 관리가 필요합니다.
추천 흐름: 먼저 CSS로 해결 → 상태/데이터가 붙으면 JS → 모션 디자이너 타임라인이 필요하면 Lottie/SVGator → 차트는 D3.

SVG 애니메이션의 대표 종류 10가지

아래 카드는 실제 서비스 UI에서 반복적으로 쓰이는 패턴입니다. 각 카드는 독립 SVG이므로 확대해도 깨지지 않고, 일부는 HTML 파일 안에서 직접 움직입니다.

01 CSS transform

아이콘·버튼 상태 전환

햄버거→닫기, 좋아요, 토글, hover 피드백에 적합합니다.

02 Stroke draw

로고·서명 그리기

stroke-dasharray/offset으로 선이 그려지는 듯한 효과를 만듭니다.

03 SMIL animate

SVG 자체 반복 애니메이션

<animate>, <animateTransform>, <animateMotion>로 SVG 내부에서 선언합니다.

04 Motion path

경로 이동

지도 경로, 배송 위치, 온보딩 여정, 과학 시뮬레이션에 좋습니다.

05 Path morph

형태 변형

플레이↔일시정지, 검색↔닫기, blob 배경처럼 형태가 바뀌는 모션입니다.

06 Mask / clip

리빌·스켈레톤

콘텐츠가 닦이듯 나타나거나 이미지 안쪽만 채워지는 효과를 구현합니다.

07 Filter

글로우·블러·고무질감

알림, 음악 비주얼라이저, 히어로 배경에 분위기를 더합니다.

08 viewBox

줌·카메라 워크

복잡한 다이어그램에서 특정 영역으로 확대 이동하는 설명 영상형 UI입니다.

09 Data viz

차트 전환

새 데이터가 들어올 때 막대, 선, 축을 부드럽게 보간합니다.

10 Lottie/SVG

툴 기반 복합 모션

After Effects에서 제작한 일러스트 타임라인을 앱/웹에 재생합니다.

인터랙티브 데모: 종류별로 눌러보기

탭을 누르면 같은 SVG 무대가 다른 목적의 애니메이션으로 바뀝니다. “이미지형 / 영상형 / 와이어프레임형” 차이를 빠르게 비교할 수 있습니다.

이미지형 SVG 애니메이션

정적인 벡터 이미지를 선 그리기, 마스크, 글로우로 살립니다. 랜딩 페이지 히어로, 브랜드 로고 리빌, 빈 상태 일러스트에 가장 많이 씁니다.

.logo path { stroke-dasharray: 620; stroke-dashoffset: 620; animation: draw 3s ease forwards; } @keyframes draw { to { stroke-dashoffset: 0; } }

실제 적용 와이어프레임: 결제 버튼/진행상태/피드백

아래는 서비스에서 바로 볼 수 있는 SVG 마이크로인터랙션 예시입니다. 버튼 상태는 JS로, 체크마크는 stroke-dashoffset으로, 로딩 스피너는 CSS transform으로 처리했습니다.

상태 전환 컨트롤

실제 앱에서는 API 요청 상태에 맞춰 idle → loading → success/error로 변합니다. SVG 내부 요소에 클래스를 붙이면 로딩, 체크, 오류 모션을 한 컴포넌트에서 처리할 수 있습니다.

접근성 팁: 자동 반복 모션은 사용자가 줄이기를 요청한 경우 멈추거나 간소화해야 합니다. 이 파일도 prefers-reduced-motion을 반영합니다.

function setPhoneState(state) { phone.className.baseVal = 'phone ' + state; label.textContent = { idle:'결제하기', loading:'처리 중', success:'완료', error:'다시 시도' }[state]; }

영상형 구성: SVG로 만드는 짧은 설명 시퀀스

SVG는 영상 파일처럼 고정 프레임을 굽는 대신, 도형과 타임라인을 브라우저가 실시간으로 그립니다. 그래서 짧은 제품 설명, 온보딩, 로딩 대기 화면은 MP4보다 작은 파일로도 충분할 때가 많습니다.

2. 경로 설명
3. 데이터 변화
4. 처리 중
5. 완료 피드백

실제 적용사례를 화면 단위로 해석하기

브랜드명보다 중요한 것은 “어떤 문제를 해결하려고 움직였는가”입니다. 아래는 현업에서 곧바로 요구사항으로 바뀌는 사례입니다.

1. 랜딩 페이지 히어로 이미지

목표: 첫 화면에서 서비스의 원리를 3초 안에 설명.

  • 선 그리기: 데이터 흐름/배송 경로/연결망 표현
  • 마스크 리빌: 제품 스크린샷이 닦이듯 등장
  • viewBox 줌: 복잡한 구조에서 핵심 기능으로 카메라 이동

2. 앱 온보딩/빈 상태

목표: 사용자가 다음 행동을 이해하도록 시각적 맥락 제공.

  • Lottie/SVG renderer: 디자이너의 After Effects 타임라인을 재생
  • 반복은 2~3초 이하로 가볍게
  • title/desc 또는 aria-label로 의미 제공

3. 결제·저장·업로드 피드백

목표: 기다림, 성공, 오류를 명확히 안내.

  • CSS spinner: 요청 중 상태
  • stroke-dashoffset 체크: 성공 완료
  • shake + 색 전환: 오류와 재시도 유도

4. 데이터 대시보드

목표: 숫자 변화의 방향과 크기를 즉시 인지.

  • D3 transition으로 막대/선/축 보간
  • 새 데이터 로딩 시 skeleton shimmer
  • 차트 포커스 영역은 clipPath로 강조

5. 지도·물류·모빌리티

목표: 이동 경로와 현재 위치를 직관적으로 전달.

  • animateMotion: 객체가 경로를 따라 이동
  • dash animation: 아직 지나가지 않은 경로와 완료 경로 구분
  • 카메라 이동: viewBox 또는 그룹 transform

6. 교육·기술 문서 다이어그램

목표: 단계와 원리를 순서대로 설명.

  • 회로, 파이프라인, API 흐름을 path draw로 설명
  • 클릭 시 단계별 레이어 표시
  • 정적인 PNG보다 확대 품질과 수정성이 좋음

실무 적용 체크리스트

디자인 단계

도형 이름을 의미 있게 정하고, 애니메이션을 “장식”이 아니라 “상태 전달”로 정의합니다. 반복 모션은 사용자가 오래 보는 화면에서 피로감을 줄 수 있으므로 짧고 멈출 수 있게 설계합니다.

// 권장 네이밍 예시 svg#payment-feedback g#loading-spinner path#success-check g#error-cross path#route-progress

개발 단계

SVG는 인라인으로 넣어야 내부 path, rect, circle을 CSS/JS로 제어하기 쉽습니다. 파일로만 불러오면 DOM 내부에 접근하기 어렵습니다.

@media (prefers-reduced-motion: reduce) { .animated { animation: none; } } svg[role="img"] > title + desc { /* 의미 있는 SVG에는 title/desc를 둡니다 */ }
