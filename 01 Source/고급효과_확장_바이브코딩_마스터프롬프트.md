## 0. 핵심 원칙 (모든 작업에 적용)

1. 기존 CSS Lab·SVG Lab·공통 컴포넌트·테마를 **삭제하거나 깨뜨리지 않는다.**
2. 새 기능은 각각 **독립적인 Lab 섹션(해시 라우트 + 사이드바 항목)**으로 추가한다.
3. 기존 공통 컴포넌트(`createDemoFrame` 등)와 디자인 토큰을 **반드시 재사용**한다. 없으면 최소 구현하되 기존 스타일을 따른다.
4. 모든 데모는 **미리보기 + 컨트롤 + 설명 + 복사 가능한 코드 + 초기화**를 포함한다.
5. **컨트롤 값과 표시 코드는 동일 params에서 파생**(정직한 코드). 둘이 어긋나면 버그.
6. 애니메이션은 `transform`·`opacity`·`stroke-dashoffset` 중심. `width/top/left/margin` 애니메이션 금지(나쁜 예 비교 데모 제외).
7. **`prefers-reduced-motion`을 존중**하고, 2장의 **전역 모션 설정 허브**와 연동한다.
8. **모든 `requestAnimationFrame` 루프·`ResizeObserver`·`IntersectionObserver`·이벤트 리스너·오디오 노드·3D 리소스는 Lab을 떠날 때 반드시 정리(cleanup/dispose)** 한다.
9. **성능 규율**: 보이지 않는 탭/화면 밖에서는 RAF 루프를 자동 일시정지한다(`document.visibilityState` + `IntersectionObserver`).
10. 외부 라이브러리는 **꼭 필요한 곳만 CDN 비동기 로드**(이 확장에서는 **3D만 Three.js**). 로드 실패 시 graceful fallback.
11. 모바일 반응형 필수. 코드 주석은 **모두 한국어**.
12. 각 Lab 구현 후 **직접 실행 가능한 검수 체크리스트**를 표로 보고한다.

## 2. 공통 확장 — 전역 모션 설정 허브 (★ 다른 Lab보다 먼저 ★)

> 이 허브를 먼저 만들어야 이후 모든 Lab이 "접근성·일시정지·속도"를 한곳에서 공유한다. 기존 `store`를 **확장**한다(새로 만들지 말 것).

```text
기존 전역 store를 확장해서 "전역 모션 설정 허브(MotionSettings)"를 만들어줘.
새 Lab 섹션 "Accessibility & Motion Hub"도 추가해줘. 기존 store/테마/컴포넌트를 재사용해.

확장할 상태(기존 paused/speed에 추가/정리):
- systemPrefersReducedMotion : matchMedia('(prefers-reduced-motion: reduce)') 결과
- reducedMotionOverride      : "system" | "reduce" | "no-preference"
- effectiveReducedMotion     : 위 둘을 합산한 최종 boolean (파생값)
- animationSpeed             : 0 | 0.25 | 0.5 | 1 | 1.5 | 2   (기존 speed와 통합)
- pauseAllAnimations         : boolean (기존 paused와 통합)
- disableAutoplay            : boolean
- disableParallax            : boolean
- disablePageTransitions     : boolean
- flashSafetyMode            : boolean (깜빡임 차단)

동기화 규약:
- localStorage에 저장하고 새로고침 후 복원.
- 루트(:root 또는 <html>)에 data 속성/클래스 반영:
  data-reduced-motion, data-animation-speed, data-pause-animations, class="rm-sim"/"motion-paused"
- CSS 변수 제공: --motion-speed-multiplier, --motion-duration-fast/normal/slow, --speed
- 전역 CSS 규칙:
  [data-pause-animations="true"] *{ animation-play-state:paused !important; }
  [data-reduced-motion="true"] *{ /* 큰 transform 이동/반복 축소, opacity 중심 */ }
  단, 사용자 조작에 필요한 :focus-visible 전환까지 제거하진 않는다.

헬퍼 함수(다른 Lab이 공유):
- getMotionDuration(baseMs, settings)      // reduced/speed 반영한 실제 duration
- shouldAutoplay(settings)                  // disableAutoplay/reduced 반영
- shouldReduceMotion(settings)
- getPlaybackRate(settings, localRate)      // speed × localRate

Accessibility Hub UI(기존 createDemoFrame/Control 재사용):
- 시스템 상태 표시(OS prefers-reduced-motion)
- Reduced Motion Override 라디오: Follow System / Reduce / Allow
- Animation Speed 셀렉트
- Pause All Animations / Disable Autoplay / Disable Parallax / Disable Page Transitions / Flash Safety 토글
- 데모: ① reduced 전/후 카드 비교 ② 키보드 내비(Tab+Enter/Space, 포커스 가시화, 미니 모달 focus trap)
  ③ :focus-visible(마우스 vs 키보드 차이) ④ aria-live(polite/assertive 선택) ⑤ Flash Safety(위험 효과는 명시적 preview 버튼 후 짧게만)
- 코드 패널: prefers-reduced-motion CSS / matchMedia JS / 전역 적용 스니펫

검수 기준:
1. OS prefers-reduced-motion 감지 + Override 반영(effectiveReducedMotion).
2. 설정이 localStorage에 저장·복원된다.
3. 루트 data 속성/CSS 변수가 적용된다.
4. Pause All이 기존 CSS/SVG 데모의 반복 애니메이션을 멈춘다.
5. 키보드 내비/aria-live 데모가 동작한다.
6. 기존 테마 토글·데모가 깨지지 않는다.
```

## 3. 추가 Lab의 공통 마운트 규약 (각 Lab이 따를 계약)

각 Lab은 다음을 지킨다(중복 코드 방지):
- 진입 시 `mount()`, 이탈 시 `unmount()`를 노출하는 **모듈 객체**로 작성. `unmount()`에서 RAF/observer/listener/오디오/3D 리소스를 **전부 정리**.
- 무한 모션 영역엔 `data-animated="true"`(전역 pause 대상).
- `store.subscribe()`로 MotionSettings 변화를 구독해 즉시 반영.
- 컨트롤 값 → (a) 스테이지 반영 + (b) `buildCode(params)`로 코드 패널 갱신.
- 화면 밖이면 `IntersectionObserver`로 루프 일시정지, 보이면 재개.

## 부록 C — 프롬프트 작성 황금 규칙
1. **목표를 한 문장으로 먼저** — "무엇을 만들지" 명확히.
2. **스택 고정** — 본 앱은 "바닐라 단일 HTML + 3D만 CDN". 매번 상기.
3. **숫자로 구체화** — "빠르게"보다 "0.3초", "많이"보다 "입자 200개".
4. **단계를 쪼개기** — [기본]→[개선] 누적식. 한 번에 다 시키지 않기.
5. **검증 요청** — 각 Lab 끝 "검수 기준"을 스스로 점검해 표로 보고.
6. **느낌 조절은 변수로** — duration·easing·intensity를 컨트롤/CSS 변수로 빼 튜닝 쉽게.
7. **cleanup 강박** — RAF·observer·listener·audio·3D 리소스는 반드시 정리.
