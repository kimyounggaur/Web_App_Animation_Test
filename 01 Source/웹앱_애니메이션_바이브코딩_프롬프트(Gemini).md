# [Vibe Coding Prompt] 고급 웹 인터랙션 및 시각 효과 마스터 랩 (Advanced Web Effects Master Lab)

## 1. 프로젝트 개요 및 프롬프트 목적
* **목표:** 앞서 구축한 CSS/SVG 모션 랩을 넘어, 현대 웹 프론트엔드에서 구현할 수 있는 가장 고도화된 5가지 시각/청각적 효과를 한곳에서 실험하고 코드를 확인할 수 있는 **'고급 웹 인터랙션 플레이그라운드(Advanced Web Effects Master Lab)'**를 개발합니다.
* **대상 AI 엔진:** Cursor, Claude Code, GitHub Copilot 등 자연어로 빌드하는 모든 AI 에이전트 전용.
* **아키텍처 제어:** 개발의 편의성과 Vibe Coding의 직관성을 위해, 외부 무거운 빌드 도구(Vite, Webpack 등) 없이 브라우저에서 바로 실행 가능한 **단일 HTML 파일(Self-contained HTML) 형태로 빌드**하도록 지시합니다. 필요한 외부 라이브러리는 공인된 고성능 CDN(cdnjs, unpkg)을 통해 비동기 로드합니다.

## 2. 디자인 시스템 및 UI/UX Vibe (노션 미니멀)
AI가 UI를 설계할 때 아래의 디자인 가이드라인을 엄격히 준수하도록 세팅합니다.

* **컨셉:** 극도로 정갈하고 군더더기 없는 **노션(Notion) 모바일 및 웹 인터페이스 스타일**.
* **테마 및 색상 시스템:**
  * **Light 모드 (기본):** 배경 `#FFFFFF`, 카드/서페이스 `#F7F7F5`, 테두리 `#E9E9E7`, 주 텍스트 `#37352F`, 부 텍스트 `#787774`
  * **Dark 모드:** 배경 `#191919`, 카드/서페이스 `#202020`, 테두리 `#2F2F2F`, 주 텍스트 `#E3E3E2`, 부 텍스트 `#9A9A97`
  * **브랜드 포인트 컬러 (노션 블루):** `#2383E2` (인터랙션 및 활성화 유도용)
* **공간 설계:** 섹션별 여백(Padding/Gap)을 `24px` 이상으로 넉넉하게 배치하여 복잡한 시각 효과가 노출될 때 화면이 조잡해 보이지 않도록 제어합니다.
* **컴포넌트 구조:** 상단 고정(Sticky) 형태의 5단 탭 메뉴 네비게이션을 배치하여 사용자가 원하는 효과 섹션으로 매끄럽게 스크롤 혹은 전환되도록 합니다.

## 3. 5대 핵심 기능별 상세 엔지니어링 지침 (AI 지시문)

AI가 각 섹션을 구현할 때 반영해야 할 기술 사양과 인터랙션 정의입니다.

### 탭 1: 3D 그래픽 스페이스 (WebGL / Three.js)
* **기술 스택:** `Three.js` (CDN 로드: `https://cdnjs.cloudflare.com/ajax/libs/three.js/r128/three.min.js`)
* **데모 사양:** 뷰포트 내에 고정된 `canvas` 영역을 생성하고, 부드럽게 회전하는 미니멀한 3D 입체 기하학 도형(예: 반투명한 노션 스타일 매트 재질의 큐브 또는 토러스)을 렌더링합니다.
* **인터랙션:** 1. 마우스 커서의 움직임(또는 터치 드래그)에 따라 3D 오브젝트가 실시간으로 반응하여 가볍게 회전 각도가 변하는 인터랙티브 매핑 구현.
  2. 오브젝트의 색상, 회전 속도, 와이어프레임 모드를 실시간으로 온/오프할 수 있는 하단 슬라이더 및 체크박스 제어판 배치.

### 탭 2: 가상 물리 엔진 & 스프링 모션 (Physics & Springs)
* **기술 스택:** 라이브러리 없이 구현하는 고성능 물리 법칙 계산(오일러 적분법 기반 수식) 또는 가벼운 물리 시뮬레이션 JS 코드 적용.
* **데모 사양:** 화면 중앙에 '드래그 가능한 노션 스타일 카드 컴포넌트'를 배치합니다.
* **인터랙션:**
  1. 사용자가 마우스로 카드를 잡고 끌다가(Drag) 임의의 위치에 놓았을 때(Drop), 딱딱하게 멈추는 것이 아니라 실제 고무줄이나 용수철에 매달린 것처럼 **중력과 마찰력, 탄성력(Overshoot & Bounce)**이 계산되어 원래의 중심점으로 통통 튀며 복귀하는 물리 기반 모션 구현.
  2. 장력(Stiffness)과 감쇠비(Damping)를 조절하는 슬라이더를 제공하여 물리값 변화를 체감하게 함.

### 탭 3: 캔버스 셰이더 & 왜곡 연출 (Canvas & Shaders)
* **기술 스택:** HTML5 `Canvas 2D API`를 극한으로 최적화하거나 단순 WebGL 단일 컨텍스트 활용.
* **데모 사양:** 카드 레이아웃 안에 흐릿하고 몽환적인 그라데이션 이미지 래퍼를 배치합니다.
* **인터랙션:**
  1. 마우스 커서가 이미지 위를 지나갈 때(Mousemove), 커서 주변의 픽셀들이 물결치듯 일렁이거나 물방울이 튀기는 듯한 **리퀴드 왜곡(Liquid Distortion) 효과**를 실시간 연출.
  2. 혹은 수백 개의 미세한 파티클(점)들이 마우스 커서를 피해 도망쳤다가 마우스가 떠나면 다시 원래 이미지 형태로 결합하는 모래시계 인터랙션 구현.

### 탭 4: 청각 피드백 시스템 (Web Audio API)
* **기술 스택:** 브라우저 내장 `Web Audio API` (외부 MP3 파일 의존성 없이 오실레이터로 실시간 가상 음원 합성).
* **데모 사양:** 오디오 제어 보드 디자인 인터페이스 구현.
* **인터랙션:**
  1. **토글/스위치 모션:** 켜고 끌 때 실제 아날로그 스위치 소리(Short High-frequency Sine Wave Click)를 합성해 재생.
  2. **성공/완료 버튼:** 클릭 시 청량한 청각 보상을 주는 코드(Arpeggio Major Chord Chime) 자동 생성 및 재생.
  3. **슬라이더 제어:** 주파수(Hz)와 볼륨 값이 웹오디오 오실레이터 노드에 다이렉트로 매핑되어 시각적 조작과 청각적 변화가 완벽히 싱크로되도록 설계.

### 탭 5: 스크롤 가로 패럴랙스 (Advanced Scroll Storytelling)
* **기술 스택:** `GSAP` 및 `ScrollTrigger` (CDN 로드) 또는 Vanilla JavaScript `Scroll` 이벤트 리스너 기반 최적화 뷰.
* **데모 사양:** 사용자가 마우스 휠을 아래로 스크롤할 때, 화면은 아래로 내려가는 대신 **오른쪽으로 거대하게 밀려나는 '가로 스크롤(Horizontal Scroll)' 타임라인 영역**을 구축합니다.
* **인터랙션:** 가로로 긴 트랙 안에서 배경 오브젝트들과 텍스트 카드가 서로 다른 속도로 움직여 입체적인 공간감을 주는 시차(Parallax) 레이어 효과 구현.

## 4. 단계별 실행 계획 (Step-by-Step Build Instructions)

AI 에이전트가 코드를 작성할 때 이 순서대로 청크(Chunk)를 나누어 개발하도록 유도합니다.

* **단계 1: 전역 마크업 및 노션 테마 스타일링**
  * 모바일 뷰에서도 완벽히 대응하는 반응형 그리드 및 탭 레이아웃을 잡고, 라이트/다크 모드 변수 전환 시스템 구축.
* **단계 2: Three.js 및 물리 스프링 스크립트 작성**
  * CDN 라이브러리를 안전하게 로드하고, 초기화 셋업 진행. 3D 렌더러와 드래그 앤 드롭 물리 좌표 연산 루프(`requestAnimationFrame`) 구현.
* **단계 3: 셰이더 왜곡 및 웹 오디오 오실레이터 결합**
  * 이미지 왜곡 마우스 리스너 최적화(데반운싱 또는 쓰로틀링 적용하여 60fps 유지), 브라우저 오디오 정책(User Gesture 인터랙션 후 AudioContext 활성화)을 고려한 오디오 노드 설계.
* **단계 4: 가로 스크롤 시나리오 및 코드 뷰어 탑재**
  * 스크롤 컨테이너 바인딩. 각 효과 뷰 하단에 노션 코드 블록 디자인 스타일로 복사 가능한 실시간 소스코드 리드아웃(`pre`, `code`) 영역 구현.

## 5. 예외 처리, 성능 및 접근성 제약조건 (Strict Quality Rules)
* **성능 무조건 사수:** WebGL 및 Canvas 연산 시 CPU 부하를 방지하기 위해 사용자가 해당 탭을 보고 있지 않을 때는 렌더링 루프(`requestAnimationFrame`)가 자동으로 일시정지(Pause)되도록 보이지 않는 탭 제어 로직을 넣을 것.
* **접근성(a11y) 대응:** 전정기관 질환자나 시스템 움직임 최소화 요청 유저를 위해 `@media (prefers-reduced-motion: reduce)` 환경일 경우, 3D 회전 속도를 0으로 만들고 가로 패럴랙스를 일반 세로 스크롤 구조로 자동 fallback 처리할 것.
* **오디오 정책 준수:** 유저가 화면의 아무 곳이나 한 번 이상 '클릭'하기 전에는 Web Audio API가 에러를 내지 않고 대기 상태(`suspended`)를 유지하다가 첫 인터랙션 시 부드럽게 재개(`resume`)되도록 예외 코드를 반드시 삽입할 것.
