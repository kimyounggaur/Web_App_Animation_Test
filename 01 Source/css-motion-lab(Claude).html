Part 01

3가지 핵심 메커니즘

CSS 애니메이션은 결국 셋의 조합입니다. transition(상태가 바뀔 때 반응), @keyframes(타임라인 위에서 스스로 재생), 그리고 둘이 실제로 움직이는 대상인 transform. 여기에 움직임의 ‘느낌’을 결정하는 타이밍 함수가 더해집니다.

① Transition — 상태 전환

hover / focus / class 토글

시작 상태와 끝 상태 ‘둘’ 사이를 부드럽게 잇습니다. 트리거(주로 :hover나 JS의 클래스 변경)가 있어야 작동하고, 트리거가 풀리면 자동으로 되돌아옵니다.

// 박스에 마우스를 올려보세요
hover me
.box {
  transition: transform .35s cubic-bezier(.34,1.56,.64,1),
              background .35s ease,
              border-radius .35s ease;
}
.box:hover {
  transform: translateY(-14px) scale(1.08) rotate(-4deg);
  background: hotpink;
  border-radius: 50%;
}

② @keyframes — 타임라인 애니메이션

자동 재생 · 무한 반복 가능

0%~100% 구간에 여러 단계를 정의해 스스로 재생됩니다. 트리거가 필요 없고 무한 반복(infinite)할 수 있어 로더·강조·앰비언트 모션에 씁니다.

spinrotate 360°
pulsescale + opacity
bouncetranslateY
shaketranslateX + rotate
fliprotateY (3D)
gradientbackground-position
@keyframes spin { to { transform: rotate(360deg) } }
@keyframes pulse {
  0%,100% { transform:scale(1); opacity:1 }
  50%      { transform:scale(1.25); opacity:.55 }
}
.loader { animation: spin 1s linear infinite; }
/* animation: name | 시간 | 타이밍 | 반복 | 방향 | fill-mode */

③ Transform — 움직임의 재료

GPU 가속 · 60fps

위 둘이 실제로 ‘무엇을’ 움직이느냐. translate·scale·rotate·skew는 레이아웃을 다시 계산하지 않아 가장 부드럽습니다. 슬라이더로 직접 조합해 보세요.

// 실시간 transform
transform
0px
0°
1.0
0°
.target { transform: translateX(0px) rotate(0deg) scale(1) skewX(0deg); }

④ Timing function — 가속/감속의 ‘성격’

같은 거리, 다른 느낌

같은 1.3초·같은 거리라도 타이밍 함수가 인상을 좌우합니다. ‘재생’을 눌러 다섯 곡선을 동시에 비교해 보세요.

linear
ease-in
ease-out
ease-in-out
spring 🪄
transition-timing-function: linear;       /* 일정 속도 — 로더, 진행률 */
transition-timing-function: ease-in;      /* 천천히 출발 — 화면에서 사라질 때 */
transition-timing-function: ease-out;     /* 빨리 출발·부드러운 정지 — 등장(가장 자연스러움) */
transition-timing-function: cubic-bezier(.68,-.6,.32,1.6); /* 스프링 — 통통 튀는 느낌 */
Part 02

마이크로 인터랙션

버튼·토글·좋아요처럼 ‘손끝의 반응’을 만드는 작은 모션들. 사용자에게 “네 동작이 먹혔어”라는 피드백을 주고 제품의 완성도를 끌어올립니다. 전부 눌러볼 수 있습니다.

버튼 호버 3종

lift(들어올림) · sweep(채움) · glow(발광) — 전부 :hover만으로

클릭 리플 (Material)

클릭 지점에서 원이 퍼지는 안드로이드/머티리얼 시그니처

좋아요 하트

탭하면 팝 + 파티클이 터집니다 (인스타/트위터 패턴)

❤️

토글 스위치 & 햄버거→X

상태 변화를 모션으로 — 켜짐/열림이 한눈에

로딩 버튼 — 상태 시퀀스

기본 → 처리 중(스피너) → 완료(체크) 로 매끄럽게 전환되는 비동기 액션 패턴

// JS: 클릭하면 클래스로 상태를 갈아끼우고, CSS transition이 전환을 그린다
btn.addEventListener('click', () => {
  btn.classList.add('loading');                 // 라벨 → 스피너
  setTimeout(() => btn.classList.replace('loading','done'), 1500); // → 체크
  setTimeout(() => btn.classList.remove('done'), 2800);   // 원위치
});
Part 03

UI 컴포넌트 와이어프레임

실제 화면에 들어가는 컴포넌트 단위 모션입니다. 어떤 앱에서 쓰이는지 함께 적어 두었어요. 버튼을 눌러 동작을 확인해 보세요.

스켈레톤 → 콘텐츠 (shimmer)

데이터 로딩 중 흐르는 빛 효과. Facebook·LinkedIn·YouTube·Slack이 빈 화면 대신 쓰는 방식

민
민지 디자이너
5분 전 · 프로덕트팀

스켈레톤은 “곧 여기에 콘텐츠가 뜬다”는 신호를 주어 체감 대기 시간을 줄입니다.

모달 등장

오버레이 페이드 + 카드 scale-up. iOS·웹앱 다이얼로그의 기본 (ease-out + 살짝 오버슈트)

변경사항을 저장할까요?

오버레이는 opacity로, 카드는 scale·translateY로 등장합니다. 두 transition에 약간의 delay 차이를 주면 더 자연스럽습니다.

토스트 알림

가장자리에서 슬라이드-인 후 자동 사라짐. Gmail·Slack·안드로이드 스낵바의 비차단 알림

아코디언 펼침

FAQ·설정 화면. 높이를 모르는 콘텐츠도 grid-template-rows: 0fr→1fr로 부드럽게

CSS만으로 애니메이션이 되나요?

네. transition과 @keyframes로 대부분의 UI 모션을 자바스크립트 없이 만들 수 있습니다.

언제 JS가 필요한가요?

스크롤 위치 감지, 클릭 좌표, 물리 기반 모션, 복잡한 시퀀스 제어가 필요할 때입니다.

성능이 걱정돼요.

transform과 opacity만 애니메이션하면 60fps를 유지하기 쉽습니다. width·top·margin은 피하세요.

탭 — 미끄러지는 인디케이터

밑줄이 활성 탭으로 슬라이드. transform으로 위치/너비를 옮겨 부드럽게

인디케이터의 left/width를 직접 바꾸는 대신 translateX와 scaleX(또는 width)를 transition하면 리플로우 없이 매끄럽습니다.

패널은 fade-up으로 교체합니다. 콘텐츠 높이가 다르면 컨테이너에 min-height를 두거나 측정해 맞춥니다.

접근성: 탭은 role=tab, 패널은 role=tabpanel, 키보드 화살표 이동을 함께 구현하세요.

진행률 바

아는 진행률은 width 채움, 모르는 작업은 무한 루프(인디터미넌트)
