보고, 만지고, 복사하는 CSS·SVG 애니메이션 도감

모든 데모는 순수 CSS 또는 네이티브 SVG로 움직입니다. 컨트롤을 바꾸면 화면과 코드가 함께 바뀌고, 복사 버튼은 빈 HTML에서도 재사용할 수 있는 실제 코드를 제공합니다.

Transition

상태 사이를 부드럽게

hover, focus, class 변경처럼 두 상태를 연결할 때 사용합니다.

@keyframes

시간표가 있는 반복

로딩, shimmer, pulse처럼 스스로 반복되는 모션에 적합합니다.

Transform

60fps 우선 움직임

translate, scale, rotate, opacity를 중심으로 레이아웃 흔들림을 줄입니다.

Scroll/View

위치와 화면 전환

스크롤 진행률, route swap, 목록에서 상세로 이어지는 전환을 다룹니다.

SVG

선, 경로, 마스크

stroke drawing, animateMotion, filter, viewBox를 라이브로 확인합니다.

A11y/Perf

줄이기와 측정

prefers-reduced-motion, pause, FPS, focus-visible을 기본값으로 둡니다.

  • 목적이 피드백이면 transition, 반복이면 keyframes, 데이터/상태면 JS 제어.
  • 성능은 transform, opacity, stroke-dashoffset을 우선하고 width/top/left는 피합니다.
  • 접근성은 토글이 아니라 기본값입니다. motion pause와 reduced-motion을 항상 제공합니다.
  • 복사 코드는 프레임워크 없이 붙여 넣어도 돌아가야 합니다.

Module B

마이크로 인터랙션

클릭, focus, toggle 같은 작은 입력에 즉각적인 피드백을 줍니다.

클릭 리플

ClickFeedback

클릭 좌표에서 원이 퍼지며 입력이 접수되었음을 알려줍니다.

적용: 모바일 버튼, 리스트 아이템, 터치 UI.

button.addEventListener('click', (event) => {
  const ripple = document.createElement('span');
  ripple.className = 'ripple';
  ripple.style.left = `${event.offsetX}px`;
  ripple.style.top = `${event.offsetY}px`;
  button.append(ripple);
  ripple.addEventListener('animationend', () => ripple.remove());
});

좋아요 하트 pop

SVG 엔진 비교

CSS vs SMIL vs WAAPI vs Lottie 설명

CSSSMILWAAPI

외부 라이브러리 없이 세 가지는 실제 동작, Lottie는 개념 카드로 비교합니다.

선택: 단순 반복은 CSS/SMIL, 상태와 데이터 제어는 JS, AE 타임라인은 Lottie를 검토합니다.

SMIL: SVG 파일 안에서 <animate>, <animateMotion>으로 단독 재생
JS(WAAPI): element.animate()로 상태, 입력, 데이터와 연결
Lottie: After Effects JSON 렌더링이 필요할 때 선택하되 용량과 접근성을 검토

Stroke drawing

getTotalLength

실제 path 길이를 측정해 dasharray와 dashoffset에 넣습니다.

적용: 로고, 서명, 성공 체크, 지도 경로.

function drawPath(path) {
  const length = path.getTotalLength();
  path.style.strokeDasharray = length;
  path.style.strokeDashoffset = length;
  path.getBoundingClientRect();
  path.style.transition = 'stroke-dashoffset 1400ms ease';
  path.style.strokeDashoffset = 0;
}

대시보드 갱신

Count-upChart

숫자와 차트가 같은 상태 변경에 반응합니다.

128

적용: KPI 카드, 관리자 홈, 매출 리포트.

function animateCount(node, from, to) {
  const start = performance.now();
  requestAnimationFrame(function tick(now) {
    const p = Math.min(1, (now - start) / 500);
    node.textContent = Math.round(from + (to - from) * p);
    if (p < 1) requestAnimationFrame(tick);
  });
}

1. 상태 사이 전환인가? → transition
2. 스스로 반복되는가? → @keyframes

4. 선이 그려지거나 경로를 따라가나? → SVG stroke / animateMotion
5. 디자이너 타임라인을 그대로 써야 하나? → Lottie 검토

Recipe Library

복사 가능한 레시피 15+

빠르게 가져다 쓸 수 있는 코드 패턴입니다. 각 카드의 키워드는 위 데모의 복사 코드와 연결됩니다.

1

Button hover

150~220ms transform + shadow.

transition: transform 180ms ease;
2

Card lift

translateY + scale로 제품 카드 강조.

transform: translateY(-10px) scale(1.02);
3

Spinner

진행률이 아닌 대기 상태.

animation: spin .9s linear infinite;
4

Skeleton shimmer

데이터 도착 전 레이아웃 힌트.

transform: translateX(160%);
5

Modal enter

opacity + translateY + scale.

transform: translateY(0) scale(1);
6

Toast enter

짧은 상태 피드백.

opacity: 1; transform: translateY(0);
7

Image zoom

wrapper overflow hidden 유지.

img:hover { transform: scale(1.13); }
8

Clip reveal

before/after와 공개 효과.

clip-path: circle(48% at 50% 50%);
9

Video overlay

play 상태와 progress 표시.

transform-origin: left;
10

Scroll reveal

IntersectionObserver fallback.

entry.target.classList.toggle('is-visible');
11

View Transition

지원하면 startViewTransition.

document.startViewTransition(update);
12

SVG stroke

getTotalLength로 정확한 선 길이.

path.getTotalLength();
13

SVG motion path

경로를 따라 마커 이동.

<animateMotion> 또는 offset-path
14

결제 성공 체크

상태머신 + stroke drawing.

idle → loading → success → error
15

Reduced motion

움직임 줄이기 사용자 선호 존중.

@media (prefers-reduced-motion: reduce)
