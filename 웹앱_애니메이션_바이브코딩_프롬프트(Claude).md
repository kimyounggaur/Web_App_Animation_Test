# 🎨 웹앱 애니메이션/효과 구현 — 바이브코딩 프롬프트 모음

> SVG · CSS 애니메이션 외에 추가할 수 있는 **9가지 고급 효과**를, AI 코딩 도구(Claude, Cursor, Copilot 등)에 **그대로 붙여넣어 사용**할 수 있도록 단계별 프롬프트로 정리한 문서입니다.

---

## 📌 사용법

1. 각 섹션의 **`[0단계] 공통 컨텍스트`** 프롬프트를 먼저 입력해 환경을 맞춥니다.
2. 그다음 **`[1단계] → [2단계] → ...`** 순서대로 프롬프트를 하나씩 입력합니다.
3. 결과가 마음에 안 들면 각 섹션 끝의 **`🔧 개선 프롬프트`** 를 사용합니다.
4. `{ }` 안의 값은 본인 프로젝트에 맞게 **반드시 교체**하세요.

### ✅ 모든 프롬프트에 공통으로 붙이면 좋은 문장
```
- 코드는 복사-붙여넣기만 하면 동작하도록 완전한 형태로 작성해줘.
- 각 단계마다 "왜 이렇게 하는지" 한 줄 주석을 달아줘.
- 에러가 날 수 있는 부분은 미리 예외 처리해줘.
- 내가 수정해야 할 부분은 // TODO 로 표시해줘.
```

## 1️⃣ JavaScript 애니메이션 라이브러리 (GSAP / Framer Motion / Anime.js)

### [0단계] 공통 컨텍스트
```
나는 {React 18 + Vite / 순수 HTML+JS} 환경에서 웹앱을 만들고 있어.
지금부터 GSAP를 사용해 부드러운 UI 애니메이션을 단계별로 추가할 거야.
각 단계는 이전 단계 코드 위에 누적되도록 작성해줘.
```

### [1단계] 설치 & 초기 세팅
```
1. GSAP를 설치하는 명령어를 알려줘 (npm 기준).
2. ScrollTrigger, Flip 플러그인을 함께 등록하는 초기화 코드를 작성해줘.
3. gsap.registerPlugin() 위치와 import 순서를 명확히 표시해줘.
```

### [2단계] 기본 진입 애니메이션
```
페이지 로드 시 다음 순서로 나타나는 애니메이션을 만들어줘:
- 헤더: 위에서 아래로 fade-in (0.6초)
- 제목: 0.2초 지연 후 글자별로 stagger 등장
- 카드 3개: 0.1초 간격으로 아래에서 위로 슬라이드 인
gsap.timeline()으로 작성하고, ease는 "power3.out"을 사용해줘.
```

### [3단계] 인터랙션 애니메이션
```
다음 마이크로 인터랙션을 추가해줘:
- 버튼 hover 시: scale 1.05 + 그림자 강조 (0.3초)
- 버튼에서 마우스가 떠나면: 원래 상태로 복귀
- 카드 클릭 시: 살짝 눌리는 효과 (scale 0.95 → 1)
React라면 useRef와 useGSAP 훅을 사용하고, 메모리 누수 방지를 위한 cleanup도 넣어줘.
```

### 🔄 대안: Framer Motion (React 전용)
```
같은 효과를 GSAP 대신 Framer Motion으로 다시 작성해줘.
motion 컴포넌트, variants, whileHover, whileTap, AnimatePresence를
활용하고, 스프링(spring) 물리 효과를 적용해줘.
```

### 🔧 개선 프롬프트
```
- 애니메이션이 너무 빠르니/느리니 전체 속도를 {1.5}배로 조절해줘.
- 모바일에서는 애니메이션을 줄이고 prefers-reduced-motion을 존중하게 해줘.
```

## 2️⃣ Canvas 애니메이션 (고성능 직접 제어)

### [0단계] 공통 컨텍스트
```
HTML5 Canvas로 60fps 애니메이션을 직접 구현할 거야.
requestAnimationFrame 기반의 game loop 구조로 작성하고,
화면 리사이즈와 고해상도(레티나) 디스플레이에도 대응되게 해줘.
```

### [1단계] 캔버스 기본 구조
```
1. 전체 화면을 채우는 canvas 셋업 코드를 작성해줘.
2. devicePixelRatio를 반영해 선명하게 렌더링되게 해줘.
3. window resize 시 자동으로 캔버스 크기가 조정되게 해줘.
4. start() / stop() 로 제어 가능한 애니메이션 루프를 만들어줘.
```

### [2단계] 입자 시스템 만들기
```
화면에 떠다니는 입자(particle) 100개를 만들어줘:
- 각 입자는 위치(x,y), 속도(vx,vy), 크기, 투명도를 가짐
- 벽에 닿으면 반대 방향으로 튕김
- requestAnimationFrame 안에서 매 프레임 위치 업데이트
Particle 클래스로 구조화하고, 입자 개수는 상단 상수로 쉽게 바꿀 수 있게 해줘.
```

### [3단계] 입자 연결선 효과 (constellation)
```
입자들 사이 거리가 {120px} 이내일 때 두 입자를 선으로 연결해줘.
거리가 가까울수록 선이 진해지게 투명도를 조절하고,
마우스 위치 주변의 입자는 마우스를 향해 살짝 끌려오게 해줘.
```

### 🔧 개선 프롬프트
```
- 성능을 위해 입자 개수를 화면 크기에 비례해 자동 조절해줘.
- 색상 테마를 {네온 블루 계열}로 바꿔줘.
```

## 3️⃣ WebGL & 3D (Three.js)

### [0단계] 공통 컨텍스트
```
Three.js로 인터랙티브 3D 씬을 만들 거야.
{React Three Fiber / 순수 Three.js} 로 진행하고,
씬·카메라·렌더러·조명·애니메이션 루프 구조를 명확히 분리해줘.
```

### [1단계] 기본 3D 씬 구성
```
1. Three.js 설치 명령어를 알려줘.
2. 검은 배경에 회전하는 3D 큐브 하나를 화면에 렌더링해줘.
3. PerspectiveCamera, AmbientLight + DirectionalLight를 배치해줘.
4. OrbitControls로 마우스 드래그 회전/줌이 가능하게 해줘.
5. 창 크기 변경에 대응하게 해줘.
```

### [2단계] 재질과 조명 고급화
```
큐브를 다음으로 업그레이드해줘:
- 금속 질감(MeshStandardMaterial, metalness/roughness 조절)
- 환경광 반사를 위한 간단한 environment map
- 부드러운 그림자(shadow) 활성화
- 바닥 평면(plane) 추가해서 그림자가 비치게 해줘
```

### [3단계] 인터랙션 & 애니메이션
```
- 마우스를 따라 카메라가 부드럽게 시점 이동(parallax)하게 해줘.
- 큐브에 마우스를 올리면 색이 변하게(raycaster 사용) 해줘.
- 큐브 클릭 시 통통 튀는 애니메이션을 재생해줘.
GSAP 또는 lerp 보간으로 부드럽게 처리해줘.
```

### 🔄 대안: Babylon.js
```
같은 3D 씬을 Babylon.js로 작성해줘. Babylon 특유의 PBR 재질과
내장 GUI, 파티클 시스템을 활용한 버전으로 보여줘.
```

### 🔧 개선 프롬프트
```
- 모델 파일({.glb})을 불러와서 큐브 대신 표시하는 코드로 바꿔줘.
- 성능 모니터(stats.js)를 추가해줘.
```

## 5️⃣ Scroll-Triggered 애니메이션

### [0단계] 공통 컨텍스트
```
스크롤에 반응하는 애니메이션을 만들 거야.
가벼운 효과는 Intersection Observer로, 정밀한 효과는 GSAP ScrollTrigger로
구현하고 싶어. 두 방식을 모두 보여줘.
```

### [1단계] Intersection Observer 기본
```
스크롤해서 요소가 화면에 들어오면 fade-in + 위로 슬라이드되게 해줘.
- 한 번만 실행되도록(unobserve) 처리
- threshold와 rootMargin 의미를 주석으로 설명
- 여러 요소에 .reveal 클래스만 붙이면 자동 적용되는 구조로 만들어줘
```

### [2단계] GSAP ScrollTrigger 패럴럭스
```
ScrollTrigger로 다음 효과를 만들어줘:
- 배경 이미지가 스크롤보다 느리게 움직이는 패럴럭스
- 섹션 진입 시 텍스트가 좌우에서 등장
- scrub:true로 스크롤 위치와 애니메이션을 동기화
- 각 섹션에 markers를 켜서 디버깅 가능하게 해줘
```

### [3단계] 고급: Pin & 가로 스크롤
```
- 특정 섹션을 화면에 고정(pin)하고 그 안에서 가로로 슬라이드되게 해줘.
- 스크롤 진행도에 따라 진행 바(progress bar)가 차오르게 해줘.
- 숫자가 0에서 목표값까지 카운트업되는 효과를 추가해줘.
```

### 🔧 개선 프롬프트
```
- 모바일에서 스크롤 성능 저하가 없도록 will-change와 throttle을 적용해줘.
- 새로고침 시 스크롤 위치가 꼬이지 않게 ScrollTrigger.refresh()를 처리해줘.
```

## 6️⃣ 입자 / 배경 효과 (PixiJS / 인터랙티브 배경)

### [0단계] 공통 컨텍스트
```
웹앱 배경에 인터랙티브한 입자 효과를 깔 거야.
{PixiJS(2D 고성능) / tsParticles(간편)} 로 구현하고,
콘텐츠 위에 z-index로 깔끔하게 배치되게 해줘.
```

### [1단계] tsParticles로 빠르게 (초보 추천)
```
1. tsParticles 설치 명령어를 알려줘.
2. 전체 화면 배경에 떠다니는 별/눈 입자 효과를 만들어줘.
3. 마우스를 올리면 입자가 반발(repulse)하게 해줘.
4. 색상/개수/속도를 설정 객체에서 쉽게 바꿀 수 있게 해줘.
```

### [2단계] PixiJS로 커스텀 (고급)
```
PixiJS로 다음을 만들어줘:
- WebGL 렌더러로 입자 {2000개}를 부드럽게 렌더링
- 입자들이 사인파를 따라 물결치듯 움직임
- 마우스 주변에 빛나는 글로우 효과
- BlurFilter로 부드러운 배경 분위기
```

### [3단계] 콘텐츠와 결합
```
- 입자 배경 위에 반투명 카드 UI를 올려 가독성을 확보해줘.
- 다크모드/라이트모드에 따라 입자 색이 자동 전환되게 해줘.
- 페이지 이탈 시 애니메이션을 정지해 CPU를 아끼게 해줘.
```

### 🔧 개선 프롬프트
```
- 저사양 기기에서 입자 수를 줄이는 fallback을 넣어줘.
- 입자가 마우스 클릭 지점에서 폭발하듯 퍼지는 효과를 추가해줘.
```

## 7️⃣ 물리 기반 애니메이션 (스프링 / 물리 엔진)

### [0단계] 공통 컨텍스트
```
실제 물리처럼 자연스럽게 움직이는 애니메이션을 만들 거야.
가벼운 건 스프링 보간으로, 본격적인 충돌/중력은 {Matter.js(2D) / Rapier(고성능)}
물리 엔진으로 구현하고 싶어.
```

### [1단계] 스프링 보간 (라이브러리 없이)
```
요소를 마우스 커서로 부드럽게 따라오게 해줘:
- 스프링 공식(stiffness, damping)을 직접 구현
- 목표 위치까지 탄력 있게 도달 후 살짝 오버슈트
- requestAnimationFrame 루프로 매 프레임 갱신
값을 바꿔가며 느낌을 조절할 수 있게 변수로 빼줘.
```

### [2단계] Matter.js 2D 물리 세계
```
Matter.js로 다음을 만들어줘:
1. 설치 명령어 안내
2. 중력이 작용하는 2D 공간 생성
3. 위에서 떨어지는 공 여러 개 + 바닥/벽 충돌
4. 마우스로 물체를 집어서 던질 수 있게(MouseConstraint) 해줘
5. 공끼리 부딪히면 자연스럽게 튕기게 해줘
```

### [3단계] UI에 물리 적용
```
- 버튼이나 아이콘을 물리 객체로 만들어 클릭하면 떨어지게 해줘.
- 글자 하나하나가 물리 블록이 되어 무너지는 텍스트 효과를 만들어줘.
- 화면을 흔들면(또는 버튼 클릭 시) 모든 물체가 튀어오르게 해줘.
```

### 🔄 대안: React Spring
```
React 환경이라면 위 스프링 효과를 react-spring(@react-spring/web)으로
다시 작성해줘. useSpring, useTrail, config.wobbly를 활용해줘.
```

## 8️⃣ 모션 감지 (기기 자이로스코프 / 센서)

### [0단계] 공통 컨텍스트
```
모바일 기기의 기울기·움직임에 반응하는 효과를 만들 거야.
DeviceOrientation / DeviceMotion API를 사용하고,
iOS 13+의 권한 요청 처리도 반드시 포함해줘.
```

### [1단계] 권한 요청 & 데이터 읽기
```
1. iOS에서 필요한 권한 요청 버튼과 코드를 작성해줘
   (DeviceOrientationEvent.requestPermission).
2. 기기의 alpha/beta/gamma 값을 실시간으로 화면에 표시해줘.
3. 센서를 지원하지 않는 기기를 위한 안내 메시지를 넣어줘.
```

### [2단계] 기울기 패럴럭스
```
기기를 기울이면 배경 레이어들이 서로 다른 속도로 움직이는
패럴럭스 효과를 만들어줘:
- 기울기 값을 부드럽게 보간(smoothing)
- 너무 민감하지 않게 감도(sensitivity) 조절 변수 제공
- 데스크톱에서는 마우스 위치로 대체 동작하게 해줘
```

### [3단계] 흔들기 감지
```
- DeviceMotion으로 기기를 흔드는 동작(shake)을 감지해줘.
- 흔들면 화면이 진동하고 효과음/햅틱(navigator.vibrate)이 울리게 해줘.
- 오작동 방지를 위한 임계값(threshold)과 쿨다운을 넣어줘.
```

### 🔧 개선 프롬프트
```
- HTTPS가 아니면 센서가 동작하지 않는다는 점을 안내 UI로 표시해줘.
- 배터리 절약을 위해 화면이 안 보일 때 센서 리스너를 해제해줘.
```

## 9️⃣ 실시간 쉐이더 효과 (GLSL Shader)

### [0단계] 공통 컨텍스트
```
WebGL 쉐이더(GLSL)로 실시간 비주얼 효과를 만들 거야.
{Three.js의 ShaderMaterial / 순수 WebGL / shadertoy 스타일} 로 진행하고,
vertex/fragment 쉐이더와 uniform 개념을 주석으로 설명해줘.
```

### [1단계] 첫 프래그먼트 쉐이더
```
1. 화면을 채우는 평면(plane)에 fragment shader를 적용하는
   Three.js 기본 코드를 작성해줘.
2. 시간(uTime), 해상도(uResolution), 마우스(uMouse) uniform을 전달해줘.
3. 시간에 따라 색이 부드럽게 변하는 그라데이션을 표현해줘.
GLSL 문법을 처음 보는 사람도 이해하게 줄마다 설명해줘.
```

### [2단계] 노이즈 & 물결 효과
```
- Simplex/Perlin 노이즈 함수를 넣어 흐르는 액체 같은 배경을 만들어줘.
- 마우스 위치에서 파동(ripple)이 퍼지게 해줘.
- 색상 팔레트를 {보라-핑크 그라데이션}으로 적용해줘.
```

### [3단계] 이미지/텍스처 후처리
```
- 이미지를 텍스처로 불러와 다음 효과를 입혀줘:
  · 마우스를 따라가는 왜곡(distortion)
  · 글리치(glitch) 효과
  · RGB 색수차(chromatic aberration)
- 마우스를 올리면 효과 강도가 커지게 해줘.
```

### 🔧 개선 프롬프트
```
- 모바일 GPU에서 끊기지 않도록 연산을 단순화한 lite 버전을 만들어줘.
- 효과 파라미터를 dat.GUI/leva로 실시간 조절할 수 있게 해줘.
```
